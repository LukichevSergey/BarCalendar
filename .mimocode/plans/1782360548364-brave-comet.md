# BarCalendar MVP — План реализации

## Что создаём
Минималистичный календарь в меню-баре macOS (аналог Itsycal). Чистый SwiftUI, никакого AppKit.

## MVP Фичи
1. Иконка с текущей датой в меню-баре
2. Выпадающий мини-календарь (месяц)
3. События из Apple Calendar (EventKit)
4. Навигация по месяцам
5. Настройка начала недели (пн/вс)

---

## Архитектура

### Структура файлов
```
BarCalendar/
├── BarCalendarApp.swift          # Точка входа, MenuBarExtra
├── CalendarState.swift           # @Observable состояние + EventKit
├── CalendarDropdownView.swift    # Контейнер выпадающего окна
├── MonthHeaderView.swift         # Заголовок месяца + стрелки навигации
├── WeekdayHeaderRow.swift        # Ряд П В С Ч П С В
├── CalendarGridView.swift        # Сетка дней месяца (7 колонок)
├── DayCell.swift                 # Ячейка дня с точкой событий
├── EventsSectionView.swift       # Список событий сегодня/завтра
├── SettingsView.swift            # Выбор начала недели
├── BarCalendar.entitlements      # Sandbox entitlements
└── Assets.xcassets/
```

### Ключевые типы
- `CalendarState` — `@Observable` класс, хранит: `displayedMonth`, `events: [EKEvent]`, `hasCalendarAccess`, `startOfWeek`
- Все View — структуры SwiftUI, получают `CalendarState` через `@Bindable`

## Реализация поэтапно

### Этап 1: Настройка проекта
- Обновить `SWIFT_VERSION` до 6.0 в `project.pbxproj`
- Создать `BarCalendar.entitlements` (sandbox)
- Добавить в build settings:
  - `INFOPLIST_KEY_LSUIElement = YES` (скрыть из Dock)
  - `INFOPLIST_KEY_NSCalendarsUsageDescription` (описание для EventKit)

### Этап 2: App Entry + State
- **`BarCalendarApp.swift`**: заменить `WindowGroup` на `MenuBarExtra` с `.window` стилем
  - Label: `Text(dayNumber)` с monospaced шрифтом (防 jitter)
  - Content: `CalendarDropdownView`
- **`CalendarState.swift`**:
  - `@Observable final class CalendarState`
  - Загрузка `startOfWeek` из UserDefaults
  - `requestCalendarAccess()` → `EKEventStore.requestFullAccessToEvents()` (async)
  - `fetchEvents()` — запрос событий за текущий отображаемый месяц
  - `previousMonth()` / `nextMonth()` — навигация + перезагрузка событий
  - `eventsForDay(_:)` — фильтрация событий по дню

### Этап 3: UI — Заголовок и подписи
- **`MonthHeaderView.swift`**: "Июнь 2026" + chevron left/right кнопки
- **`WeekdayHeaderRow.swift`**: 7 коротких названий дней с учётом `startOfWeek`

### Этап 4: UI — Сетка календаря
- **`CalendarGridView.swift`**: `LazyVGrid` с 7 колонками
  - Пустые ячейки перед 1-м числом (offset = (firstWeekday - startOfWeek + 7) % 7)
  - Дни месяца
- **`DayCell.swift`**: число + синий кружок на сегодня + оранжевая точка если есть события

### Этап 5: События
- **`EventsSectionView.swift`**:
  - Разделы "Сегодня" и "Завтра"
  - Каждое событие: цветной кружок календаря + название + время
  - Состояние "нет доступа" — подсказка перейти в Системные настройки

### Этап 6: Настройки
- **`SettingsView.swift`**: `Picker` для начала недели (Пн/Вс), `segmented` стиль
- Встроен вниз `CalendarDropdownView`

## Design решение

| Элемент | Решение |
|---------|---------|
| Иконка в меню-баре | Текст с номером дня, monospaced шрифт |
| Сегодня | Синий кружок за числом |
| События в календаре | Оранжевая точка под числом |
| События в списке | Цвет календаря + название + время |
| Ширина dropdown | 280pt |
| Нет доступа к календарю | Иконка + текст "Grant access in System Settings" |

## Gotchas

- `LSUIElement = YES` обязателен — без него приложение будет в Dock
- `requestFullAccessToEvents()` — async API для macOS 14+, заменяет deprecated `requestAccess(to:)`
- Сравнение дат для событий: `Calendar.isDate(_:inSameDayAs:)`, не `==`
- `EKEventStore` создаётся один раз, переиспользуется
- События загружаются за весь отображаемый месяц (нужны для точек в сетке)

## Верификация
1. Build проекта в Xcode — нет ошибок компиляции
2. Приложение появляется в меню-баре (не в Dock)
3. Клик по иконке → открывается dropdown с календарём
4. Сетка корректно отображает текущий месяц
5. Стрелки навигации переключают месяцы
6. При первом запуске — системный диалог разрешения календаря
7. После разрешения — события отображаются в списке
8. Настройка начала недели работает и сохраняется
